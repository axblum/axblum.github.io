SQL Injection

and Protecting your Database

Posted: 11/2/2015

What is SQL injection you ask?

SQL injection is a type of vulnerability that an application that accesses or updates an SQL database can be subject too. SQL injection can lead to the accessing or manipulating data maliciously.
SQL injection can be avoided or mitigated by following some general concepts when working with accessing data within a database via a frontend application. The first part is permissions. It is very important to make sure that your application is using the appropriate credentials to access the data or insert additional data. This means not using the same account as the one you used to setup your database.
The second thing to keep in mind is, how are the application is interacting with the back end database. Lets say you have a form that inserts a row of data into a table, it is best practice to sanitize the data before making the insert. This can be done by using one of the many frameworks that do ORM.
