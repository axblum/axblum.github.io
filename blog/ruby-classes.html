Ruby Classes

Money

Posted: 10/18/2015

This Blog post is going to be about Ruby classes and why to use them. Ruby classes are how you build object in Ruby. Lets take a real world example of an object or a thing. In this case I chose to talk about money. If you ever have taken a look at a dollar bill from your wallet. You might notice there is a whole lot of information printed on that dollar bill. Each bill that is printed by the US Treasury has a set of things that are printed on the bill itself, from the serial number of the bill, to how much the bill is worth. It's also worth mentioning that the treasury prints lots of bills, with a template, this is very much how a Ruby class works. Lets use the bill example below. 

          class Money
            def initialize(denomination,serial_number,country)
            @denomination = denomination
            @serial_number = serial_number
            end
        end
In the example above, we have created an Money class. This is like the template that the US Mint would use to make a new dollar bill.The initialize method is special, because each time a new bill is created, we call the initialize method as seen below.
            
                jackson = Money.new(100,"1234","US")
Not only can a class have instance variables that are denoted with the @ in front of their name. Instance variables allow, for individual properties to be saved with each bill that is printed, such as the serial number. Not only can classes, have properties, but they also can have actions associated with them. Lets say that for a bill to be authentic the serial number of the bill, must be 10 numbers long. We can create a method that is part of the class that would allow us to check if the bill is authentic, like so:

                def valid
                    if @serial_number.size = 10
                        true
                    else
                        false
                    end
                end
As always if you need additional information on how to use classes. I suggest you consult the Ruby-Docs
